# Design System: Kizunu

## Overview

**Creative North Star: "The Signal Lattice"**

Kizunu should feel like the visual language of an expanding open system, not the website of a single SaaS feature. The core image is a dark technical field with visible structure, bright signals, plotted movement, and modular parts that can scale into a wider ecosystem of channels, connectors, cadences, inboxes, analytics, and AI layers. The system should feel experimental, creative, and technical at the same time: authored with intent, but never precious.

This direction borrows the ecosystem discipline of the Grafana system work, the authored density of `hex.inc/info`, and the polished technical calm of `sim.ai`, then pushes them toward a more open-source, builder-facing posture. It should feel like infrastructure with a visual thesis. Color behaves like signal, composition behaves like routing, motion behaves like system reveal. The design must explicitly reject common SaaS defaults, common AI defaults, safe startup landing-page polish, sterile enterprise dashboard polish, and the generic AI startup template.

**Key Characteristics:**

- Dark technical substrate with visible grid logic
- Full-palette signal accents used as system markers
- Experimental composition held inside disciplined geometry
- Open-source posture expressed through modular, extensible visual language
- Choreographed motion that reveals structure instead of decorating it

## Colors

The palette should behave like a control surface at night: deep dark neutrals holding a small set of bright, high-contrast signal colors that feel computational, modular, and slightly industrial.

### Primary

- **Blaze Channel Orange** (`[to be resolved during implementation]`): The anchor brand signal. Use it for core calls to action, active route emphasis, key verbs, hero punctuation, and the most public expression of Kizunu's voice.

### Secondary

- **Signal Cyan** (`[to be resolved during implementation]`): Use for product logic, routing, technical detail, and moments that should feel precise, connected, and infrastructural rather than promotional.

### Tertiary

- **Protocol Magenta** (`[to be resolved during implementation]`): Use for experimental moments, AI-adjacent ideas, and compositional tension where the system needs a more synthetic, future-facing note.
- **Alert Yellow** (`[to be resolved during implementation]`): Use for warnings, highlighted metrics, or structured emphasis that should feel immediate rather than friendly.
- **Vector Lime** (`[to be resolved during implementation]`): Use sparingly for success states, active system energy, and selected moments that need a living, electrical quality.

### Neutral

- **Carbon Field** (`[to be resolved during implementation]`): The primary surface color. This is the world the system lives in.
- **Graphite Rail** (`[to be resolved during implementation]`): Use for panel separation, grids, borders, and infrastructural lines.
- **Fog Trace** (`[to be resolved during implementation]`): Use for secondary text, coordinates, labels, and quiet metadata.
- **Cold Signal White** (`[to be resolved during implementation]`): Use for primary text and high-priority interface language. It should feel slightly cooled, never pure white.

### Named Rules

**The Signal Rule.** Bright colors are never decorative gradients or ambient glow. They mark channels, states, routes, emphasis, and system events.

**The Dark Field Rule.** The dark surface is the stage. Do not wash it out with soft SaaS panels or overfill it with floating cards.

### Named Rules

**The Instrument Label Rule.** Mono belongs to labels, indices, coordinates, and short system markers. It does not become the reading voice of the whole product.

**The No-Brochure Rule.** Headings should feel authored and infrastructural, not like polished startup marketing copy dropped into a template.

## Elevation

Kizunu should use a layered system on top of an otherwise flat substrate. Depth comes first from structure, contrast blocks, borders, cropping, and motion sequencing, not from soft floating cards. If shadows appear, they should be ambient and structural, used only to clarify active state or focal priority. The system should feel built into a grid, not hovering above it.

### Named Rules

**The Panel Rule.** Surfaces sit on the lattice. They do not float like generic SaaS cards.

**The Depth-Through-Structure Rule.** Use framing, overlap, and motion to create depth before reaching for shadow.

## Do's and Don'ts

### Do:

- **Do** build layouts from visible structure: rails, grids, modules, plotted spacing, and deliberate alignment tension.
- **Do** use the full palette as a system language, with each accent color carrying a distinct role instead of acting as random decoration.
- **Do** let dark space carry tension. Some zones should feel dense and instrumented, while others stay open enough to create rhythm.
- **Do** make motion explain the system, with reveals that clarify sequence, routing, hierarchy, or state change.
- **Do** make the brand feel open-source in posture: modular, extensible, builder-facing, and precise.

### Don't:

- **Don't** use common SaaS defaults.
- **Don't** use common AI defaults.
- **Don't** build a safe startup landing page.
- **Don't** fall into sterile enterprise dashboard polish.
- **Don't** use the generic AI startup template: soft gradients, vague intelligence claims, glowing blobs, floating cards, and interchangeable product diagrams.
- **Don't** turn Kizunu into a CRM-looking brand. The system must read as infrastructure, orchestration, and ecosystem, not pipeline software with better marketing.
